. 2.1.2 Exécution des tests de non-régression Il est nécessaire d'automatiser l'exécution des tests de non-régression d'Audacy pour s'assurer qu'une modi cation du code source d'Audacy n'engendre pas de régression, c'est-à-dire qu'une fonctionnalité nouvelle ne provoque pas une perturbation de fonctions existantes par e et secondaire. Pour cela, un script shell devra être développé, dont l'objectif sera d'exécuter tous les tests de non-régression présents dans le code source d'Audacy. Etant donné que certains tests de non-régression peuvent durer longtemps, il faudra que ce script puisse se lancer en spéci ant que seuls les tests rapides sont à exécuter.
