Une dépen- dance circulaire intervient lorsqu'un premier module requiert la présence d'un second module qui requiert lui-même ce premier module. On se retrouve alors dans une situa- tion d'interblocage. Ce problème peut être résolu grâce à la création d'un graphe de dépendances. Dans d'autres langages comme Java ou C, des outils existent pour la création d'un tel graphe : JDepend [23] pour Java et CPPDepend [24] pour C++ en sont des exemples. Il est alors possible de supprimer manuellement les dépendances en s'aidant du graphe obtenu. Pour le moment, aucun outil de ce type n'a été trouvé pour Audacy, mais le problème des dépendances a été résolu en chargeant les modules de manière à séparer les modules interdépendants, en modi ant directement l'ordre dans lequel sont inclus les modules dans le code maude
